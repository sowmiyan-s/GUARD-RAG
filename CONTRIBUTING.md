# Contributing to GuardRAG

Thank you for your interest in contributing to GuardRAG! We value all types of contributions from code to documentation and bug reports.

## Getting Started

1.  **Fork the repository** on GitHub.
2.  **Clone your fork** locally.
3.  **Create a new branch** for your feature or bug fix.
4.  **Install dependencies** using the local install instructions in the README.
5.  **Make your changes** and verify them locally.
6.  **Submit a Pull Request** (PR) with a clear description of your changes.

## Development Standards

-   Maintain consistency with the existing codebase.
-   Ensure all code is private-by-design (no external telemetry or cloud dependencies).
-   Update the README if change impacts existing workflows.

---

## Code of Conduct

Please be respectful and professional in all interactions.
